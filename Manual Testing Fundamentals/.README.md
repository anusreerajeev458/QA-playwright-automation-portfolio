# Manual Testing Fundamentals: LMS QA Simulation 
## Project Overview This folder contains artifacts from a self-directed, end-to-end **QA Simulation** conducted on a **Learning Management System (LMS)**. 
Because a live production environment was not available, I acted as the solo QA Engineer to independently establish a comprehensive testing framework. I reverse-engineered business expectations and assumed realistic software requirements to build all foundational testing documentation from scratch. 
--- 
## What This Simulation Demonstrates 
* **Requirement Analysis:** Interpreted and structured ambiguous features into clear, testable criteria.
* **Test Documentation Mastery:** Created professional-grade QA deliverables following industry-standard formats.
* **Defect & Traceability Management:** Established end-to-end coverage mapping to ensure no requirement was left untested.
---
## Artifacts Included
* 📄 **LMS - Test Strategy & Plan:** Outlines the scope, environment needs, testing types (Functional, UI, Smoke), and risk mitigation strategies.
* 📄 **LMS - Test Scenario & Test Case Document:** Detailed, step-by-step test cases covering positive, negative, and boundary value scenarios for core LMS modules (User Login, Course Enrollment, Dashboard).
* 📄 **LMS - Requirement Clarification & Review Notes:** Simulates the collaborative process of identifying and resolving gaps in early product requirements.
* 📄 **LMS - Test Requirement Traceability Matrix (RTM):** Maps business requirements directly to test cases to verify 100% test coverage.
* 📄 **LMS - Test Execution Report & Defect Log:** Logs simulated execution runs, demonstrating professional defect reporting, severity/priority assignment, and tracking metrics.
* 📄 **ATM Withdrawal-Test Scenarios:** An additional module demonstrating foundational test design techniques (Equivalence Partitioning and Boundary Value Analysis).
